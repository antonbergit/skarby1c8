Обработка "Загрузка данных из Торговли для частных предпринимателей Украины базовая"

С помощью этой обработки можно загрузить данные из конфигурации Торговля для ЧП. В форме обработки следует указать файл обмена, в который были выгружены данные из конфигурации Торговля для ЧП, и нажать на кнопку Загрузить.

Важно. Перенос данных из конфигурации Торговля для ЧП возможен.

Файлы, необходимые для выгрузки данных из конфигурации Торговля для ЧП входят в поставку программы Управление небольшой фирмой для Украины (базовая), редакция 1.6 и находятся в каталоге \ConvTrCPUkrBase.

  • Перенос данных ТЧП  - УНФ 1.6.epf - внешняя обработка выгрузки данных во внешний файл из конфигурации Торговля для ЧП, которая использует для работы 1С:Предприятия 8.2.
  • УниверсальныйОбменДаннымиXML.epf - внешняя обработка универсального обмена данными в формате XML. Необходима для функционирования обработки выгрузки данных во внешний файл из конфигурации Торговля для ЧП.

Порядок действий при переносе данных

Выполните резервное копирование информационной базы конфигурации "Торговля для ЧП". Выполните все регламентные процедуры в конфигурации "Торговля для ЧП".

  • Удалите помеченные на удаление объекты.  
  • Выполните проведение документов.

Важно. На момент переноса данных в информационной базе "Торговля для ЧП" не должно быть отрицательных остатков товаров по складам и организациям, ДС по кассам. Информация об отрицательных остатках товаров, ДС не будет выгружена из информационной базы конфигурации.
  • Откройте внешнюю обработку "Перенос данных ТЧП  - УНФ 1.6.epf" с помощью пункта меню "Файл"-"Открыть".
  • При первом запуске обработки укажите путь к файлу внешней обработки универсального обмена данными в формате XML (УниверсальныйОбменДаннымиXML.epf). По умолчанию, данный файл расположен в каталоге шаблонов конфигурации УНФ, в подкаталоге \ConvTrCPUkrBase.
  • В появившемся диалоге укажите имя внешнего файла, в который будет производиться выгрузка данных. Укажите дату, на которую актуальны остатки товаров, денежных средств и взаиморасчетов.
  • В диалоговом окне появится список объектов, которые могут быть выгружены. По умолчанию будет выгружена информация по настройкам параметров учета, справочникам и остаткам, рекомендуемым к выгрузке. При необходимости можно отключить выгрузку информации по любому из справочников или отключить выгрузку остатков по любому из регистров конфигурации.
  • При выгрузке остатков задолженности клиентов можно выгрузить задолженность с учетом конкретных документов реализации или авансов, перечисленных по заказам клиентов. Будет выгружен документ "Ввод остатков" с соответствующим видом операции, с указанием суммы непогашенной задолженности. Документы по которым есть непогашенная задолженность, не выгружаются, а автоматически генирируются при загрузке.
  • После настройки правил выгрузки данных необходимо нажать на кнопку "Выгрузить". При выгрузке данных осуществляется проверка на наличие отрицательных остатков и выдается соответствующее сообщение. При необходимости эти данные необходимо откорректировать. После того, как все объекты исправлены необходимо повторно нажать на кнопку "Выгрузить". Данные будут выгружены во внешний файл.
  • Произведите установку конфигурации Управление небольшой фирмой для Украины (базовая), редакция 1.6 для 1С:Предприятия 8.2.
  • В конфигурации "Управление небольшой фирмой" сопоставьте предопределенную организацию "Наша фирма" с соответствующей организацией конфигурации Торговля для ЧП. Для этого добейтесь соответствия реквизитов организации "Наименование" в обоих информационных базах.
  • Для загрузки данных выберите раздел "Настройки" - "Инструменты администратора" - "Загрузка данных из ТЧП". В открывшемся диалоговом окне укажите внешний файл, в который была произведена выгрузка, и нажмите на кнопку "Загрузить". Данные будут загружены в информационную базу из внешнего файла.
  • После проведения загрузки убедитесь, что данные загружены корректно. Остатки товаров, денежных средств и взаиморасчетов фиксируются документом "Ввод остатков". С помощью отчетов проверьте правильность выгрузки остатков в обеих базах.
Важно. При повторной загрузке остатков данные в документах ввода остатков, созданных конвертацией, перезаполняются заново.
